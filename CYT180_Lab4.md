### Field Definitions

Although the log file is unstructured text, it contains important security-relevant fields:

| Information    | Description |
|----------------|-------------|
| Timestamp      | Date and time of the login attempt (`Jun 14 15:16:01`) |
| Hostname       | Name of the machine generating the log (`combo`) |
| Service        | SSH daemon or PAM module (`sshd(pam_unix)`) |
| Event type     | `authentication failure` indicates failed login; `session opened` indicates a successful session |
| Username       | `logname=` or `ruser=` fields (may be empty) |
| Source IP      | `rhost=<IP>` shows the origin of the login attempt |
| UID/EUID       | System user IDs of the process attempting authentication (`uid=0 euid=0`) |
| TTY            | Terminal associated with the login (`tty=NODEVssh`) |

## MapReduce Recap
MapReduce is a distributed programming model used to process large datasets across many machines.
It consists of three main phases:

### 1. Map Phase
- The **Mapper** reads input data **line by line**.
- For each line, it outputs **key–value pairs**.
- Each key represents the entity we want to group or count.
- Each value usually represents a unit of measurement (e.g., `1` for counting events).

### 2. Understanding the Input
Each line from the log file looks like this:
``` yaml
Jun 14 15:16:01 combo sshd(pam_unix)[19939]: authentication failure; logname= uid=0 euid=0 tty=NODEVssh ruser= rhost=218.188.2.4
```
Important parts:
- `"authentication failure"` → indicates a **failed SSH login**  
- `"rhost=<IP>"` → source IP address of the login attempt
- Only failed login events are counted in this lab. Successful logins or session opened lines are ignored.

### 3. Mapper Output (Key-Value Pairs)

| Mapper Input Line | Mapper Output |
|------------------|---------------|
| Log line above   | 218.188.2.4&nbsp;&nbsp;&nbsp;&nbsp;1 |

- Key = IP address (`rhost` value)  
- Value = 1 (each failed login counts as 1)


### 2. Shuffle and Sort Phase

- Hadoop automatically **groups values by key** across all mappers.
- Keys are **sorted** before being sent to the reducer.
- This ensures that **all values for a given key arrive together**.

Example after shuffle & sort:
```yaml
218.188.2.4 [1, 1, 1, ...]
192.168.1.10 [1, 1, ...]
```

### 3. Reduce Phase
- The **Reducer** processes all values associated with a key.
- Usually, it aggregates them (e.g., sums counts).
- Output is written to **HDFS** in `part-00000`, `part-00001`, etc.

| IP Address     | Failed Login Count |
|----------------|------------------|
| 218.188.2.4    | 143              |
| 192.168.1.10   | 78               |

- Reducer sums all values per IP address  
- Final output written to HDFS (`part-00000`)

### Choosing a Good Key

The **key** determines how data is grouped.

A good key:
- Represents a meaningful question
- Enables aggregation

Examples:

| Question | Key | Value |
|----------|-----|-------|
| Which IPs have the most failed logins? | IP address (`rhost`) | 1 |
| Which usernames are most targeted? | Username (`ruser` or `logname`) | 1 |
| How many failed login attempts total? | constant string (`"failed"`) | 1 |

Choosing the wrong key can make MapReduce results useless.

### Why MapReduce is Useful

- Scales to **millions of records**.
- Handles data **too large for one machine**.
- Hadoop manages **distribution, parallel execution, and fault tolerance**.
- In cybersecurity, MapReduce is perfect for analyzing large security logs, counting events, and identifying attack patterns.

## Step 3  —  Write the Mapper
In this step, we will write a **Mapper** program in Python.  
The mapper reads SSH log entries **line by line** and emits key–value pairs for **failed login attempts**.
For each log line, the mapper will:
- Check if the event is a **failed SSH login**
- Extract the **source IP address**
- Output:
  ```yaml
  IP_ADDRESS 1
  ```
This allows Hadoop to later **count failed logins per IP address**.

### 1. Understanding the Input
Each line from the log file looks like this:
``` yaml
Jun 14 15:16:01 combo sshd(pam_unix)[19939]: authentication failure; logname= uid=0 euid=0 tty=NODEVssh ruser= rhost=218.188.2.4
```

Important parts:
- `"authentication failure"` → indicates a **failed SSH login**  
- `"rhost=<IP>"` → source IP address of the login attempt
- Only failed login events are counted in this script. Successful logins or session opened lines are ignored.

### 2. Create the Mapper Script
Create a file named `mapper.py`:

```python
%%writefile mapper.py
#!/usr/bin/env python3
import sys
for line in sys.stdin:
    line = line.strip()

    # Process only failed SSH login attempts
    if "authentication failure" in line:
        ip = None
        parts = line.split()
        # Extract IP after 'rhost='
        for p in parts:
            if p.startswith("rhost="):
                ip = p.split("=")[1]
                break
        if ip:
            print(f"{ip}\t1")
```

### 3. Make the Mapper Executable
```bash
!chmod +x mapper.py
```

## Step 4  —  Write the Reducer
In this step, we will write a **Reducer** program in Python.  
The reducer aggregates all the counts emitted by the mapper for each **IP address**.
The Reducer Will:
- Read sorted key–value pairs from the mapper via **STDIN**
- Sum all values for the same key (IP address)
- Output the total failed login attempts per IP

Example input (from mapper after shuffle & sort):
```yaml
10.0.0.5 1
192.168.1.10 1
192.168.1.10 1
```
Expected output:
```yaml
10.0.0.5 1
192.168.1.10 2
```

### 1. Create the Reducer Script

Create a file named `reducer.py`:

```python
%%writefile reducer.py
#!/usr/bin/env python3
import sys

current_ip = None
current_count = 0

for line in sys.stdin:
    line = line.strip()
    if not line:
        continue

    ip, count = line.split("\t", 1)

    try:
        count = int(count)
    except ValueError:
        continue

    if current_ip == ip:
        current_count += count
    else:
        if current_ip:
            print(f"{current_ip}\t{current_count}")
        current_ip = ip
        current_count = count

# Output the last IP
if current_ip == ip:
    print(f"{current_ip}\t{current_count}")
```

## Step 5  —  Run the MapReduce Job

Now that the **mapper** and **reducer** are ready and the dataset is in HDFS, we can run the Hadoop **MapReduce job** using **Hadoop Streaming**.
Hadoop fails if the output folder already exists. Always remove it first:
```python
!hdfs dfs -rm -r /user/hadoop/output_ssh_failures || true
# The || true ensures the command does not fail if the directory does not exist.
```

### 1. Hadoop Streaming Command
```python
!$HADOOP_HOME/bin/hadoop jar $HADOOP_HOME/share/hadoop/tools/lib/hadoop-streaming-3.3.0.jar \
    -input /user/hadoop/logs/Linux_2k.log \
    -output /user/hadoop/output_ssh_failures \
    -mapper mapper.py \
    -reducer reducer.py \
    -file mapper.py \
    -file reducer.py
```
**Explanation of the command:**

| Option     | Meaning                                  |
| ---------- | ---------------------------------------- |
| `-input`   | Path in HDFS to the input file           |
| `-output`  | Path in HDFS where output will be stored |
| `-mapper`  | Python mapper script                     |
| `-reducer` | Python reducer script                    |
| `-file`    | Include local scripts with the job       |


### 2. Verify the Job Completed
After running, check the output folder:
```python
!hdfs dfs -ls /user/hadoop/output_ssh_failures
```
You should see a file like `part-00000` and `_success`
